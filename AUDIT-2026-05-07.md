## ✅ FIXES APLICADOS NESTA RODADA

### Cálculos errados (Critical/High)
1. **Bug #1 — `cascadeRecalc` calculava `custo_por_kg` 1000x menor**
   `src/services/cascadeRecalc.js:44` — alinhado com a fórmula canônica de `PreparoFormScreen` (`* 1000` quando unidade é g/mL).
2. **Bug #2 — `AtualizarPrecos` gravava preço por kg sem conversão de unidade**
   `src/screens/AtualizarPrecosScreen.js:147` — agora usa `calcPrecoBase(numericValue, qtdLiquida, row.unidade_medida)`.
3. **Bug #4 — `recalcularCombo` UPDATE-ava coluna inexistente**
   `src/services/cascadeRecalc.js:106` — removido. Combos sempre re-derivam custo client-side.
4. **Contracts #2 — `aiPricing` lia `faturamento_estimado` mas coluna é `faturamento_mensal`**
   `src/services/aiPricing.js:33` — corrigido. Sugestões de preço agora usam dado real.
5. **Bug #9 — meta CMV/Margem zero virava 35%/15% por causa de `||` falsy**
   `src/screens/HomeScreen.js:773,778` — agora usa `Number.isFinite` pra distinguir 0 de inválido.
6. **Bug #10 — Embalagens mass adjust zerava `preco_unitario` quando qtd=0**
   `src/screens/EmbalagensScreen.js:362` — mantém valor antigo se quantidade inválida.
7. **Bug #17 — `ProdutoFormScreen` não invalidava cache nem notificava após cascade**
   2 callsites — agora chama `clearQueryCache` + `notifyDataChanged` após save.

### Segurança (Critical/High/Medium)
8. **H1 — RLS migration criada para tabelas faltantes**
   `supabase/migrations/20260507000000_security_rls_missing_tables.sql` cria + ativa RLS em
   `device_tokens`, `notif_prefs`, `account_deletion_requests`. **AÇÃO MANUAL**: rodar este SQL no painel Supabase.
9. **H2 — Logout ordem invertida (signOut Supabase ANTES do reset local)**
   `src/contexts/AuthContext.js` — antes: reset local → signOut. Se signOut falhasse (rede), token ficava em storage e usuário virava zumbi. Agora: signOut primeiro; reset só se OK; em fallback de erro, força reset local pra não vazar dados.
10. **H3 — Web inactivity timeout (1h) implementado**
    `src/contexts/AuthContext.js` — antes: web ficava logado indefinidamente. Agora: listeners de mouse/teclado/visibilidade + heartbeat de 60s. Após 1h sem atividade, força signOut.
11. **M1 — `user_id` enforcement defensive em UPDATE/DELETE**
    `src/database/supabaseDb.js` — wrapper agora injeta `.eq('user_id', currentUserId)` em UPDATE/DELETE. Defense-in-depth: se RLS falhar por engano de config, esta camada bloqueia.
12. **M4 — `MatrizBCG` SQL com placeholder ao invés de interpolação**
    `src/screens/MatrizBCGScreen.js:157,158` — antes: `WHERE data = '${currentMonth}'` (interpolado). Agora: `WHERE data = ?` com param.
13. **M6 — Security headers no Vercel**
    `vercel.json` — adicionados X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, Content-Security-Policy.
14. **L5 — Logs de erro não vazam payload em produção**
    `src/database/supabaseDb.js` — `JSON.stringify(row)` agora só roda em `__DEV__`.

## ⚠️ DEBT MAPEADA — não corrigida nesta rodada

### Critical (precisa rodada dedicada)
- **Bug #3 — `delivery_combo_itens` sem coluna `unidade`** → custos de combo com preparo/MP em unidade ≠ default ficam 1000x errados. Requer migration + UI de coleta de unidade.
- **Contracts #1 — `parseWhere` em `supabaseDb.js` só aceita AND** → `dependenciesService.softDelete` pode estar quebrado silenciosamente (queries `COUNT(*)` retornam 0). Requer reescrita do parser ou troca por chamadas Supabase JS diretas.

### High
- **Bug #5 — `getDivisorRendimento` retorna 1 silenciosamente quando rendimento=0**
- **Bug #7 — `ProdutoForm.salvar` sem guard de double-tap**
- **Bug #8 — `autoSaveImmediate` em DeliveryCombos sem mutex**
- **Bug #11 — cascade dispara recalc completo a cada save** (latência ≥5s pra usuários grandes)
- **Bug #12 — DELETE+INSERT sem transação em ProdutoForm** (produto pode ficar sem ingredientes em falha de rede)
- **Contracts #3 — discriminador `tipo` sem CHECK constraint** em delivery tables
- **Contracts #4 — sem UNIQUE(user_id, nome)** permite duplicatas em insumos/produtos/categorias
- **Contracts #5 — defaults legados `'Grama(s)'`, `'Unidades'`** no schema
- **Contracts #6 — schema bifurcado** (`src/database/migration-*.sql` vs `supabase/migrations/`)
- **Contracts #7 — `runAsync` retorna `{changes: 1}` hardcoded** (UPDATE/DELETE com WHERE inexistente também retorna 1)
- **Contracts #8 — falta `unidade_uso` em `produto_ingredientes`/`preparo_ingredientes`**
- **Contracts #9 — `vendas.data TEXT` (não TIMESTAMPTZ)** + sem snapshot de preço/custo no momento da venda

### Medium
- 18 cópias inline de `safeNum` ainda nos screens (helper já existe em utils — falta trocar imports)
- AsyncStorage keys sem namespace por user (drafts vazam entre contas)
- Tipos boolean inconsistentes (INTEGER vs SMALLINT vs BOOLEAN no schema)
- 7 funções @deprecated ainda exportadas em `utils/calculations.js`
